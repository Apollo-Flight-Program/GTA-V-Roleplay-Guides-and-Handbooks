# OOC Communication Glossary

Terms players use when discussing radio procedures, communication mechanics, channel management, and communication-related RP conventions. These are out-of-character concepts — the language of players setting up and managing the game.

---

## Radio System Terms

| **Term** | Definition |
|---|---|
| **Voice chat / VC** | Real-time in-game voice communication using proximity voice or a dedicated app like TeamSpeak or Discord. |
| **Proximity voice** | Voice chat that only other nearby players can hear. Standard for in-scene RP. |
| **Radio script** | A server mod that simulates radio channels. Only players on the same frequency hear each other. |
| **Frequency / channel** | A radio channel setting in the server's radio script. |
| **Radio range** | The in-game or scripted limit on how far a radio transmission travels. |
| **Open mic** | Player's mic transmitting unintentionally. |
| **Push to talk (PTT)** | Keybind that activates voice when held down. Standard setting on most RP servers. |
| **Hotmic** | Leaving your microphone open accidentally. A form of OOC leak into IC space. |
| **Discord comms** | Using Discord calls instead of or alongside in-game radio. Common for staff, admin coordination, or OOC callouts. |
| **TeamSpeak / TS** | Third-party voice software commonly used by RP servers for radio channel simulation. |
| **CAD system** | Computer-Aided Dispatch — web-based system where LEO and EMS log calls, track units, and submit reports. |

---

## Communication RP Terms

| **Term** | Definition |
|---|---|
| **Radio RP** | Roleplaying radio communication as a character, using proper codes, callsigns, and brevity. |
| **Callsign** | Your character's official radio identifier (e.g., *"4-Adam-12"*). |
| **On the air** | Player is currently transmitting on a radio channel. |
| **Tac channel** | A secondary channel used for a specific operation; separate from primary dispatch. |
| **OOC in radio** | Breaking character while transmitting over in-game radio. Considered bad form. |
| **Breaking character on comms** | Dropping IC voice/persona while in a communication RP scene. |
| **Third-party communication** | Using out-of-game tools (phone calls, Discord) to coordinate in ways that would not be possible IC. Sometimes called *metagaming* if the info crosses IC. |

---

## Channel Organization

| **Term** | Definition |
|---|---|
| **Primary** | The main operating channel for an agency. All routine traffic. |
| **Tac / Tactical** | Secondary channel for a specific active operation. |
| **Mutual aid channel** | A designated channel for coordination between multiple agencies. |
| **Encrypted / secure channel** | A channel with restricted access; used for sensitive operations. |
| **Broadcast / all-call** | A message sent out on all agency channels simultaneously. Usually for emergencies. |
| **Silent dispatch** | Sending call information to units via MDT (mobile data terminal) without a radio broadcast — used when stealth is important. |

---

## Server Communication Rules (Common)

| **Term** | Definition |
|---|---|
| **Radio rule** | Server rule requiring all communications to be via server radio scripts, not third-party apps, for IC information. |
| **No metagaming comms** | Prohibition on using OOC communication channels to pass IC information (locations, police movements, etc.). |
| **Streamer mode** | Server option to delay or obscure certain info to prevent stream sniping. |
| **Staff radio / admin comms** | Staff-only channel for managing the server, separate from player channels. |
