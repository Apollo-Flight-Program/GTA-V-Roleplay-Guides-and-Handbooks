# Tactical Operations Reference

A reference guide for military and SWAT/tactical unit roleplay. Covers the planning cycle, formation types, breach procedures, call signs, and terminology used during tactical operations.

---

## The Tactical Planning Cycle

Before any operation, tactical units follow a planning process:

| Phase | Name | Content |
|---|---|---|
| **1** | Mission Receipt | Receive orders from command; understand the mission |
| **2** | Mission Analysis | Assess the objective, threat, terrain, and available assets |
| **3** | Course of Action Development | Develop 2–3 plans for achieving the objective |
| **4** | Rehearsal | Walk through or talk through the plan; each member knows their role |
| **5** | Execution | Conduct the operation |
| **6** | After-Action Review | Debrief; what worked, what didn't, what to improve |

---

## SMEAC Briefing Format

The standard tactical briefing follows the SMEAC format:

| Letter | Element | Content |
|---|---|---|
| **S** | Situation | Current conditions — friendly forces, enemy/threat, terrain |
| **M** | Mission | Single sentence: *Who does what, where, when, and why* |
| **E** | Execution | How the mission is carried out — phases, tasks per element |
| **A** | Administration | Logistics — ammunition, medical, vehicles, communications |
| **C** | Command | Chain of command; who leads if the primary is incapacitated |

## Tactical Zones

| Zone | Description |
|---|---|
| **Objective** | The specific target building, vehicle, or area |
| **Assault position** | Covered position just outside the objective — final staging point |
| **Breach point** | The specific entry point into the objective |
| **Support position** | Position covering the assault element; overwatch, suppression |
| **Blocking position** | Cuts off enemy escape routes |
| **Extraction point (EP)** | Where the element goes after completing the mission |
| **Rally point (RP)** | Fallback point for element consolidation if separated |

## Formation Types

### Patrol Formations (Movement)

| Formation | Use |
|---|---|
| **File (single column)** | Movement through dense terrain (jungle, alley); front/rear security |
| **Staggered column** | Road movement; alternates sides for spacing |
| **Wedge** | Open terrain; all-around security with firepower forward |
| **Line (on-line)** | Assault through open terrain; maximum forward firepower |
| **Echelon (left/right)** | Flanking; skewed line facing a threat to one side |

### Room Entry Formations

| Formation | Use |
|---|---|
| **Button hook** | Entry team splits at the door, each going to a different wall |
| **Cross** | Entry team crosses in an X pattern; both operators pass through the door together |
| **Single file** | Sequential entry through a narrow doorway; first in peels to the nearest corner |

## Breach and Clear Sequence

### Exterior Approach
1. Element moves to assault position undetected
2. Position confirmed by all elements via radio
3. Breach team moves to breach point; support element covers

### Breach
| Method | Use |
|---|---|
| **Ballistic breach** | Shooting door mechanism; fast, noisy |
| **Ram/mechanical** | Battering ram; for reinforced doors |
| **Explosive breach** | Breaching charge; extreme noise/shock; used when other methods fail |
| **Bypass** | Unlocked window, alternate entry — silent option |

### Clear
1. **Threshold evaluation** — Brief pause to assess threat immediately inside
2. **Entry** — Element moves in using chosen formation
3. **Near corner first** — Clear the threats closest to entry first
4. **Fatal funnel** — Move through the doorway quickly; do not pause in the opening
5. **Sector assignment** — Each operator is responsible for a defined sector
6. **Consolidation** — Element calls "Clear!" when room is secure; move to next room

## Radio Procedures During Operations

| Call | Meaning |
|---|---|
| **Element ready** | Unit is in position and ready to execute |
| **Execute / Go** | Order to begin the operation |
| **Breach complete** | Entry point is open; ready for entry |
| **Room clear** | Specific room is secured |
| **Objective secure** | All assigned areas are cleared |
| **Consolidate** | Bring the element together at the objective |
| **Exfil / Extract** | Time to withdraw from the objective |
| **Break contact** | Disengage from enemy and withdraw |
| **Man down** | Operator is incapacitated; medical attention required |
| **Abort / No-Go** | Mission is called off |

## Fire Control Orders

When ordering fire or coordinating suppression:

**RICED format:**
| Letter | Element |
|---|---|
| **R** | Range — approximate distance to target |
| **I** | Indication — description of target or reference point |
| **C** | Command to fire or hold |
| **E** | Effect desired — neutralize, suppress, destroy |
| **D** | Duration — how long to maintain fire |

---

## Key Tactical Terms

| Term | Meaning |
|---|---|
| **Suppression** | Firing to prevent enemy from moving or firing effectively; does not require hits |
| **Covering fire** | Fire to protect a specific unit's movement |
| **Flanking** | Moving to attack the side or rear of an enemy position |
| **Bounding overwatch** | One element moves while another provides covering fire; roles alternate |
| **Assault through** | Moving directly through the objective without stopping |
| **Deconfliction** | Ensuring friendly elements are not in each other's lines of fire |
| **Collateral damage** | Unintended harm to non-combatants or civilian property |
| **Rules of Engagement (ROE)** | The specific permissions and restrictions on when and how force may be used |
| **HOT** | Hostile on target — cleared to engage |
| **Hold fire** | Do not engage; reassess |
| **Check fire** | Cease fire immediately — possible friendly fire |
